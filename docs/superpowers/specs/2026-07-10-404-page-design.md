# 404 Page — "The beam found nothing here"

**Date:** 2026-07-10
**Status:** Approved

## Purpose

A custom 404 page for Leuchtturm Media that turns a dead end into an on-brand moment: the visitor becomes a searchlight scanning a dark stretch of coast where the requested page should have been. Concept chosen over "missing film frame" and "depth-map lost scene" alternatives.

## Files

| File | Role |
|---|---|
| `app/not-found.tsx` | Server component. Catches all unmatched URLs app-wide (Next.js root `not-found` convention). Renders the scene component. Inherits root layout: Geist fonts, grain overlay, dark theme. |
| `components/not-found-scene.tsx` | Client component (`"use client"`). All interaction, GSAP animation, mouse tracking. |

No config changes. No `global-not-found.js` (single root layout, so route-level `not-found.tsx` suffices).

## Visual design

Full-viewport (`h-svh`), no scroll. Palette and tokens from `globals.css` (`--ink`, `--fog`, `--fog-dim`, `--beam`, `--line`). Sharp corners, mono kicker style consistent with the rest of the site.

Layer stack (bottom → top):

1. **Dark base layer** — near-black (`--ink`). Contains a faint (~5% opacity) outline of the giant "404" and a thin horizon line, so the screen never reads as broken/empty.
2. **Lit reveal layer** — the same composition fully lit:
   - Mono kicker: `SIGNAL LOST — 53.8°N 8.1°E`
   - Giant "404" display type (clamp-scaled, fills most of viewport width), fog-white with beam-amber accent.
   - Body line: "The beam swept this stretch of coast. Nothing lives at this address."
   - CTA link "Back to the light" → `/`, styled like existing site CTAs.
   - This layer is masked by **two CSS `mask-image` radial-gradient layers** (default `mask-composite: add` = union):
     - **Cursor searchlight:** soft-edged radial spot at cursor position. Position driven by CSS custom properties updated via GSAP `quickTo`/quickSetter with lerp so the lamp feels heavy.
     - **Lighthouse beam spot:** travels left↔right along the horizon on a slow GSAP yoyo tween. Runs always, so touch devices (no cursor) still get the reveal; on touch the beam spot is wider.
3. **Lighthouse silhouette** — small SVG tower at the right edge with a rotating conic-gradient beam wedge and a blinking beam-amber lamp. Soft blurred fog blobs behind it (static; the sweeping beam passing over them supplies the motion).
4. **Grain** — inherited from root layout.

**Escape hatch:** wordmark + "Home" link always dimly visible top-left (not behind the mask), so no visitor is stuck on a dark page.

## Motion & accessibility

- Masks and transforms only (GPU-composited); no canvas, no layout-triggering animation.
- GSAP via existing `useGSAP` pattern (`@gsap/react`), matching other components.
- `prefers-reduced-motion: reduce`: no sweep, no cursor tracking — scene rendered statically lit (mask removed), lamp not blinking.
- Text is real DOM text (selectable, screen-reader accessible); heading order `h1` for "404" message.

## Error handling / edge cases

- No JS: dark base layer with faint 404 + always-visible Home link still navigable. (Reveal is progressive enhancement.)
- Touch devices: auto-sweeping beam alone reveals content.
- Small viewports: type scales via `clamp()`; lighthouse silhouette may hide below `sm` if it crowds the composition.

## Testing

Manual verification via dev server + Playwright:

1. Visit `/does-not-exist` → HTTP 404, scene renders.
2. Screenshot dark state and cursor-revealed state.
3. Emulate `prefers-reduced-motion` → statically lit scene.
4. Verify Home link and CTA navigate to `/`.
5. `next build` passes.
